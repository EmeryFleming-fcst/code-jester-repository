- 1.1.4 [Game Objects and Material Basics]
-- A note on Parents and Children of Game Objects:
--- It is often common practice to separate visuals from logic. So, your parent object, may just hold script components and the transform and the child objects may be the actual visual components of the object (See the chair example)
--- This is done in order to keep logic and visuals in a decoupled (separate and non-linked) state, making changing one or the other a lot easier. For example, if I wanted to change the Chair from a Chair to a Couch, I would just need
to delete/replace the visuals but all of my code and logic can stay on the parent and remain the same.

- 1.1.5 [Prefabs]
-- Something to note when modifying an object in a scene that was create from a prefab. In the Inspector view, here you see the word Prefab underneath the object information section, there is an overrides dropdown.
--- Revert All will Revert the current object to match the Prefab. Apply All will apply the object changes to the prefab itself.
--- Changed on a game object that differ from a prefab will be both bolded and marked with a blue indicator on the left hand side of the inspector.
-- It is often easier to think of the Prefab system as a Blueprint system. The prefab is the blueprint that defines how the object should be, and when you create an object from that prefab you're constructing it in the same way an Architect might construct an office building from a blueprint.
